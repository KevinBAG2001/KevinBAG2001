## Flujo de trabajo con Git

```text
main       -> versión estable o lista para entrega
qa         -> validación, pruebas y control de calidad
dev        -> integración de desarrollo
feature/*  -> nuevas funcionalidades
bugfix/*   -> corrección de errores no críticos
hotfix/*   -> correcciones urgentes sobre versiones estables
release/*  -> preparación de entregas o versiones finales
```

Este flujo permite separar desarrollo, pruebas y entregas, reduciendo errores en ramas principales y mejorando la trazabilidad de los cambios.
